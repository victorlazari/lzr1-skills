## 1. Organizational Design

### Org Design Principles

| Principle | Description |
|---|---|
| Aligned to strategy | Structure follows strategy |
| Clear accountability | Every outcome has an owner |
| Minimal dependencies | Teams can move independently |
| Right span of control | 5-8 direct reports per manager |
| Communication paths | Short paths for decisions |
| Scalable | Works at 2-3x current size |

### Organizational Models

| Model | Description | Best For |
|---|---|---|
| Functional | Grouped by discipline (eng, product, design) | Early stage (<50) |
| Divisional | Grouped by product/market | Multi-product companies |
| Matrix | Dual reporting (function + product) | Large enterprises |
| Pods/Squads | Cross-functional autonomous teams | Product companies |
| Network | Fluid, project-based | Innovation-focused |

### Scaling Transitions

| Size | Key Changes | New Roles Needed |
|---|---|---|
| 1→10 | Founders do everything | First hires in key areas |
| 10→30 | Need specialization | First managers, team leads |
| 30→100 | Need process, not just people | Directors, VP-level |
| 100→300 | Need layers, delegation | C-suite, senior directors |
| 300→1000 | Need systems, governance | SVPs, org design |
| 1000+ | Need culture at scale | Chief of Staff, transformation |

## 2. VP Engineering

### VP Engineering Responsibilities

| Area | Activities |
|---|---|
| People | Hiring, retention, performance, career growth |
| Delivery | Shipping on time, quality, predictability |
| Process | Agile practices, CI/CD, engineering excellence |
| Culture | Engineering values, collaboration, innovation |
| Strategy | Technical roadmap input, resource allocation |
| Cross-functional | Alignment with product, design, business |

### Engineering Health Metrics

| Category | Metrics |
|---|---|
| Delivery | Sprint velocity, cycle time, deployment frequency |
| Quality | Bug escape rate, incident frequency, tech debt ratio |
| People | Retention, engagement, hiring pipeline |
| Efficiency | Cost per feature, infrastructure cost per user |
| Innovation | % time on new vs maintenance, patents/papers |

### VP Eng vs CTO

| Aspect | VP Engineering | CTO |
|---|---|---|
| Focus | People, process, delivery | Technology, architecture, vision |
| Orientation | Inward (team) | Outward (market, technology) |
| Metrics | Velocity, retention, quality | Innovation, technical strategy |
| Reports to | CEO or CTO | CEO or Board |
| Key skill | Management, leadership | Technical vision, architecture |

## 3. COO Operations

### COO Responsibilities

| Area | Scope |
|---|---|
| Operations | Business processes, efficiency, scaling |
| Cross-functional | Alignment between departments |
| Execution | Turning strategy into results |
| People ops | HR, culture, organizational health |
| Finance ops | Budget management, financial operations |
| Legal/Compliance | Risk management, regulatory |

### Operational Cadence

| Cadence | Activity | Participants |
|---|---|---|
| Daily | Leadership standup | C-suite |
| Weekly | Executive team meeting | All executives |
| Bi-weekly | Department reviews | COO + department heads |
| Monthly | All-hands, board prep | Full company / Board |
| Quarterly | OKR review, planning | Leadership team |
| Annually | Strategic planning, budgeting | Board + leadership |

## 4. Chief of Staff

### Chief of Staff Functions

| Function | Activities |
|---|---|
| Strategic projects | Lead cross-functional initiatives |
| Meeting management | Agenda, follow-ups, decisions |
| Information flow | Synthesize, prioritize, distribute |
| Decision support | Research, analysis, options |
| Communication | Draft communications, presentations |
| Operational rhythm | Maintain cadence, accountability |
| Special projects | M&A, fundraising support, board prep |

### Executive Meeting Framework

```
Before:
- Agenda distributed 24 hours ahead
- Pre-reads shared
- Decision items clearly marked

During:
- Timekeeper enforced
- Decisions documented in real-time
- Action items captured with owners and dates

After:
- Notes distributed within 24 hours
- Action items tracked
- Follow-up scheduled for open items
```

## 5. Leadership Development

### Leadership Competencies

| Competency | Description | Development |
|---|---|---|
| Strategic thinking | See the big picture, plan ahead | Strategy projects, board exposure |
| People leadership | Inspire, develop, retain talent | Coaching, 360 feedback |
| Execution | Deliver results consistently | Stretch projects, accountability |
| Communication | Clear, compelling messaging | Presentation coaching, writing |
| Decision-making | Make good decisions quickly | Decision frameworks, mentoring |
| Influence | Persuade without authority | Cross-functional projects |
| Resilience | Handle adversity, bounce back | Coaching, peer support |

### Executive Hiring

| Step | Activity | Timeline |
|---|---|---|
| Define | Role scope, success criteria, culture fit | 1-2 weeks |
| Source | Executive search, network, referrals | 2-4 weeks |
| Screen | Initial conversations, fit assessment | 2-3 weeks |
| Interview | Deep dives, case studies, references | 2-3 weeks |
| Decide | Team input, final decision | 1 week |
| Close | Offer, negotiation, acceptance | 1-2 weeks |
| Onboard | 90-day plan, relationships, quick wins | 90 days |

### 90-Day Executive Onboarding

| Phase | Focus | Activities |
|---|---|---|
| Days 1-30 | Learn | Listen, observe, build relationships |
| Days 31-60 | Assess | Identify gaps, opportunities, quick wins |
| Days 61-90 | Act | Implement changes, set direction, deliver wins |
