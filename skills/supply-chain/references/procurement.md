## 1. Procurement Process

### Procurement Lifecycle

| Phase | Activities | Output |
|---|---|---|
| Identify | Define need, specifications, budget | Requirements document |
| Source | Market research, identify suppliers | Supplier shortlist |
| Evaluate | RFP/RFQ, demos, references | Evaluation scorecard |
| Negotiate | Terms, pricing, SLAs | Negotiated terms |
| Contract | Legal review, sign agreement | Executed contract |
| Onboard | Integration, training, access | Active vendor |
| Manage | Performance monitoring, reviews | Vendor scorecard |
| Renew/Exit | Renewal negotiation or transition | Decision |

### Procurement Categories (Tech Company)

| Category | Examples | Annual Spend |
|---|---|---|
| SaaS/Software | Tools, platforms, licenses | High |
| Cloud infrastructure | AWS, GCP, Azure | Very high |
| Professional services | Consultants, contractors | Medium-High |
| Hardware | Laptops, servers, peripherals | Medium |
| Marketing | Agencies, ad spend, events | Medium-High |
| Facilities | Office, co-working, utilities | Medium |
| Insurance | D&O, cyber, general liability | Low-Medium |
| Legal | Law firms, compliance tools | Low-Medium |

## 2. Vendor Selection

### RFP Structure

| Section | Content |
|---|---|
| Company overview | Who we are, what we need |
| Requirements | Functional, technical, compliance |
| Evaluation criteria | How we'll score responses |
| Timeline | Key dates, decision timeline |
| Pricing format | How to structure pricing |
| Questions | Specific questions to answer |
| References | Required customer references |
| Terms | Standard terms, deal-breakers |

### Vendor Evaluation Scorecard

| Criterion | Weight | Assessment Method |
|---|---|---|
| Functional fit | 25% | Requirements coverage |
| Technical fit | 20% | Architecture, integration |
| Pricing | 20% | TCO analysis |
| Vendor viability | 10% | Company health, roadmap |
| Support | 10% | SLAs, responsiveness |
| Security/Compliance | 10% | Certifications, posture |
| References | 5% | Customer feedback |

## 3. Contract Management

### Key Contract Terms

| Term | Description | Negotiation Priority |
|---|---|---|
| Pricing | Cost structure, discounts | High |
| Term length | Contract duration | Medium |
| SLAs | Performance guarantees | High |
| Data ownership | Who owns the data | Critical |
| Termination | Exit clauses, notice period | High |
| Liability | Caps, indemnification | High |
| Auto-renewal | Automatic renewal terms | Medium |
| Price escalation | Annual increase caps | Medium |
| Audit rights | Right to audit vendor | Medium |

### SLA Framework

| Metric | Typical Target | Remedy |
|---|---|---|
| Uptime | 99.9% (8.7 hrs downtime/year) | Service credits |
| Response time | P1: 15 min, P2: 1 hr, P3: 4 hr | Escalation |
| Resolution time | P1: 4 hr, P2: 8 hr, P3: 24 hr | Credits |
| Data recovery | RPO: 1 hr, RTO: 4 hr | Contractual remedy |

## 4. Negotiation

### Negotiation Strategies

| Strategy | Description | When to Use |
|---|---|---|
| Multi-year commitment | Longer term for better pricing | Stable, proven vendors |
| Volume consolidation | Bundle spend for discounts | Multiple products/services |
| Competitive leverage | Use alternatives as leverage | Multiple viable options |
| Timing | End of quarter/year pressure | Vendor fiscal deadlines |
| Phased commitment | Start small, grow with discounts | New vendors, uncertain needs |
| Payment terms | Upfront vs monthly vs annual | Cash flow optimization |

### Negotiation Preparation

| Element | Preparation |
|---|---|
| BATNA | Best alternative if negotiation fails |
| Walk-away point | Minimum acceptable terms |
| Target | Ideal outcome |
| Leverage | What gives us negotiating power |
| Concessions | What we can give up |
| Research | Vendor's pricing, competitors, margins |

## 5. Vendor Risk Management

### Risk Categories

| Category | Risks | Mitigation |
|---|---|---|
| Financial | Vendor bankruptcy, acquisition | Financial health monitoring |
| Operational | Service outages, quality issues | SLAs, redundancy |
| Security | Data breach, compliance failure | Security assessments |
| Concentration | Over-reliance on single vendor | Multi-vendor strategy |
| Geopolitical | Sanctions, trade restrictions | Geographic diversification |
| Contractual | Lock-in, unfavorable terms | Exit clauses, data portability |

### Vendor Risk Assessment

| Factor | Low Risk | Medium Risk | High Risk |
|---|---|---|---|
| Data access | No customer data | Metadata only | Full customer data |
| Integration | Standalone | API integration | Deep integration |
| Criticality | Nice-to-have | Important | Mission-critical |
| Alternatives | Many options | Few options | No alternatives |
| Vendor size | Large, stable | Mid-market | Startup |
