## 1. Research Methods

### Method Selection Guide

| Method | Type | When to Use | Sample Size |
|---|---|---|---|
| User interviews | Qualitative, generative | Understand needs, motivations | 5-8 per segment |
| Usability testing | Qualitative, evaluative | Test designs, find issues | 5-7 participants |
| Surveys | Quantitative, generative | Measure attitudes at scale | 100+ responses |
| Card sorting | Qualitative, generative | Information architecture | 15-30 participants |
| Tree testing | Quantitative, evaluative | Validate navigation | 50+ participants |
| A/B testing | Quantitative, evaluative | Compare design variants | Statistical significance |
| Diary studies | Qualitative, generative | Understand behavior over time | 10-15 participants |
| Contextual inquiry | Qualitative, generative | Observe real environment | 4-8 participants |
| Heuristic evaluation | Expert, evaluative | Quick usability assessment | 3-5 evaluators |
| Analytics review | Quantitative, evaluative | Understand behavior patterns | All users |

### Research Planning

| Phase | Activities | Output |
|---|---|---|
| Define | Research questions, objectives, scope | Research brief |
| Plan | Method selection, recruitment, protocol | Research plan |
| Execute | Conduct sessions, collect data | Raw data, recordings |
| Analyze | Code, theme, synthesize | Findings, patterns |
| Report | Communicate insights, recommendations | Research report |
| Act | Prioritize, design solutions | Design decisions |

## 2. User Interviews

### Interview Structure

```
1. Introduction (5 min)
   - Explain purpose, consent, recording
   - Build rapport, set expectations

2. Warm-up (5 min)
   - Background questions
   - Current behavior and context

3. Core exploration (30-40 min)
   - Open-ended questions about topic
   - Follow-up probes for depth
   - Specific scenarios and examples

4. Wrap-up (5 min)
   - Anything else to share?
   - Thank participant
```

### Question Types

| Type | Purpose | Example |
|---|---|---|
| Open-ended | Explore broadly | "Tell me about how you..." |
| Behavioral | Past actions (not hypothetical) | "Walk me through the last time you..." |
| Follow-up | Dig deeper | "What do you mean by...?" |
| Contrast | Understand preferences | "How does this compare to...?" |
| Projection | Ideal state | "If you could change one thing..." |

### Questions to Avoid

| Bad Pattern | Problem | Better Alternative |
|---|---|---|
| Leading | Biases response | Neutral framing |
| Hypothetical | Unreliable predictions | Ask about past behavior |
| Yes/No | Limits depth | Open-ended questions |
| Double-barreled | Confusing | One question at a time |
| Jargon | Confusion | Plain language |

## 3. Usability Testing

### Test Planning

| Component | Description | Example |
|---|---|---|
| Objectives | What to learn | "Can users complete checkout?" |
| Tasks | Realistic scenarios | "Buy a blue shirt in size M" |
| Success criteria | Measurable outcomes | Completion rate, time, errors |
| Participants | Representative users | 5-7 per round |
| Prototype fidelity | Level of detail | Low/medium/high fidelity |

### Moderated vs Unmoderated

| Aspect | Moderated | Unmoderated |
|---|---|---|
| Facilitator | Present, guides session | Absent, automated |
| Depth | Deep probing, follow-ups | Surface-level, task-based |
| Cost | Higher (time, scheduling) | Lower (scale) |
| Speed | Slower (sequential) | Faster (parallel) |
| Best for | Complex tasks, early design | Validation, large samples |
| Tools | Zoom, Lookback | UserTesting, Maze, Lyssna |

### Usability Metrics

| Metric | Description | Benchmark |
|---|---|---|
| Task success rate | % completing task | >78% (industry average) |
| Time on task | Duration to complete | Compare to expert time |
| Error rate | Mistakes per task | <3 errors per task |
| SUS score | System Usability Scale | >68 (above average) |
| NPS | Net Promoter Score | >50 (excellent) |
| CSAT | Customer Satisfaction | >4.0/5.0 |

## 4. Synthesis and Analysis

### Affinity Mapping Process

1. Capture observations on individual notes (one insight per note)
2. Group related notes into clusters
3. Name each cluster (theme)
4. Identify relationships between themes
5. Prioritize themes by frequency and impact

### Research Outputs

| Output | Audience | Content |
|---|---|---|
| Personas | Team-wide | User archetypes, goals, pain points |
| Journey maps | Design + product | End-to-end experience with emotions |
| Insight report | Stakeholders | Key findings, evidence, recommendations |
| Opportunity map | Product + design | Prioritized problems to solve |
| Design principles | Design team | Guidelines derived from research |

### Persona Template

```
Name: [Archetype name]
Photo: [Representative image]
Quote: "[Characteristic quote from research]"

Demographics: Age, role, tech comfort
Goals: What they're trying to achieve (3-5)
Pain points: Current frustrations (3-5)
Behaviors: How they currently solve the problem
Context: Environment, constraints, tools
Needs: What would help them succeed
```

## 5. Research Operations

### Research Repository

| Component | Purpose | Tools |
|---|---|---|
| Study library | All past research | Dovetail, EnjoyHQ, Notion |
| Insight database | Searchable findings | Tagged, linked to evidence |
| Participant panel | Recruit efficiently | Panel management tools |
| Templates | Consistent quality | Protocols, consent forms |
| Metrics | Track research impact | Studies conducted, decisions influenced |

### Democratizing Research

- Train non-researchers to conduct basic studies
- Create self-service research templates
- Share findings broadly (research newsletters, lunch-and-learns)
- Build insight repositories searchable by anyone
- Establish research review processes for quality
