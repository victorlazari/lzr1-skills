# Software Engineering Recommended Reading (2024-2026)

## Essential Books (35)

### System Design & Architecture
1. **Designing Data-Intensive Applications** — Martin Kleppmann (2017), O'Reilly. The definitive guide to distributed data systems.
2. **System Design Interview Vol. 1 & 2** — Alex Xu (2020, 2022). Practical system design with real-world examples.
3. **A Philosophy of Software Design** — John Ousterhout (2nd ed, 2021). Complexity management and clean design.
4. **Clean Architecture** — Robert C. Martin (2017), Pearson. Dependency rules and architectural boundaries.
5. **Fundamentals of Software Architecture** — Mark Richards & Neal Ford (2020), O'Reilly. Architecture styles and patterns.
6. **Software Architecture: The Hard Parts** — Neal Ford et al. (2021), O'Reilly. Trade-off analysis for distributed architectures.
7. **Building Microservices** — Sam Newman (2nd ed, 2021), O'Reilly. Microservice design and deployment.
8. **Domain-Driven Design** — Eric Evans (2003), Addison-Wesley. Strategic and tactical DDD patterns.
9. **Head First Software Architecture** — Raju Gandhi et al. (2024), O'Reilly. Visual, accessible architecture learning.
10. **The Backend Engineering Handbook** — (2025). Architecture, APIs, distributed systems, scalability.

### Software Craftsmanship
11. **The Pragmatic Programmer** — David Thomas & Andrew Hunt (20th Anniversary, 2019). Timeless engineering wisdom.
12. **Clean Code** — Robert C. Martin (2008), Pearson. Code readability and maintainability.
13. **Refactoring** — Martin Fowler (2nd ed, 2018). Improving existing code design.
14. **Working Effectively with Legacy Code** — Michael Feathers (2004). Strategies for untested code.
15. **Code Complete** — Steve McConnell (2nd ed, 2004). Comprehensive construction reference.

### Backend & Distributed Systems
16. **Distributed Systems** — Maarten van Steen & Andrew Tanenbaum (4th ed, 2023). Academic distributed systems.
17. **Understanding Distributed Systems** — Roberto Vitillo (2nd ed, 2022). Practical distributed systems guide.
18. **Database Internals** — Alex Petrov (2019), O'Reilly. Storage engines and distributed data.
19. **Streaming Systems** — Tyler Akidau et al. (2018), O'Reilly. Stream processing fundamentals.

### Frontend & Web
20. **The Road to React** — Robin Wieruch (2024 Edition). Modern React with Hooks.
21. **Learning Patterns** — Lydia Hallie & Addy Osmani (2022). Design and rendering patterns for web.
22. **High Performance Browser Networking** — Ilya Grigorik (2013, still relevant). Network performance.

### Go
23. **Learning Go** — Jon Bodner (2nd ed, 2024), O'Reilly. Idiomatic Go programming.
24. **The Go Programming Language** — Donovan & Kernighan (2015). The authoritative Go reference.
25. **Concurrency in Go** — Katherine Cox-Buday (2017), O'Reilly. Go concurrency patterns.
26. **The Power of Go: Tools** — John Arundel (2024). Practical Go tool building.

### Rust
27. **The Rust Programming Language** — Klabnik & Nichols (3rd ed, 2024), No Starch Press. Official Rust book.
28. **Rust in Action** — Tim McNamara (2021), Manning. Systems programming with Rust.
29. **Programming Rust** — Blandy, Orendorff, Tindall (2nd ed, 2021), O'Reilly. Comprehensive Rust reference.
30. **Zero To Production In Rust** — Luca Palmieri (2022). Building production services in Rust.

### Performance & Operations
31. **Systems Performance** — Brendan Gregg (2nd ed, 2020), Pearson. Enterprise and cloud performance.
32. **BPF Performance Tools** — Brendan Gregg (2019), Addison-Wesley. Linux observability.
33. **Site Reliability Engineering** — Google (2016), O'Reilly. SRE principles and practices.
34. **The Site Reliability Workbook** — Google (2018), O'Reilly. Practical SRE implementation.
35. **Release It!** — Michael Nygard (2nd ed, 2018). Design for production resilience.
