## 1. GDPR (General Data Protection Regulation)

### GDPR Principles

| Principle | Description | Practical Application |
|---|---|---|
| Lawfulness, fairness, transparency | Legal basis, clear communication | Privacy notice, consent forms |
| Purpose limitation | Collect for specified purposes only | Document purposes, don't repurpose |
| Data minimization | Only collect what's necessary | Audit data fields, remove excess |
| Accuracy | Keep data correct and current | Update mechanisms, correction rights |
| Storage limitation | Don't keep longer than needed | Retention schedules, auto-deletion |
| Integrity and confidentiality | Appropriate security measures | Encryption, access controls |
| Accountability | Demonstrate compliance | Documentation, DPIAs, records |

### Legal Bases for Processing

| Legal Basis | When to Use | Example |
|---|---|---|
| Consent | Freely given, specific, informed | Marketing emails, cookies |
| Contract | Necessary to perform a contract | Delivering purchased service |
| Legal obligation | Required by law | Tax records, employment law |
| Vital interests | Protect someone's life | Emergency medical situations |
| Public interest | Public authority functions | Government services |
| Legitimate interest | Balanced business need | Fraud prevention, security |

### Data Subject Rights

| Right | Description | Response Time |
|---|---|---|
| Access (Art. 15) | Copy of their personal data | 30 days |
| Rectification (Art. 16) | Correct inaccurate data | 30 days |
| Erasure (Art. 17) | Delete their data ("right to be forgotten") | 30 days |
| Restriction (Art. 18) | Limit processing | 30 days |
| Portability (Art. 20) | Receive data in machine-readable format | 30 days |
| Object (Art. 21) | Object to processing | Without undue delay |
| Automated decisions (Art. 22) | Not be subject to solely automated decisions | 30 days |

## 2. CCPA/CPRA (California)

### CCPA vs GDPR Comparison

| Aspect | GDPR | CCPA/CPRA |
|---|---|---|
| Scope | EU residents' data | California residents' data |
| Applicability | Any org processing EU data | Revenue >$25M, 100K+ consumers, or 50%+ revenue from data |
| Legal basis | Required (6 bases) | Not required (opt-out model) |
| Consent | Opt-in (default) | Opt-out (default) |
| Private right of action | Limited | Data breaches |
| Fines | Up to 4% global revenue | $2,500-$7,500 per violation |
| DPO required | Yes (in some cases) | No (but privacy team recommended) |

### CCPA Consumer Rights

| Right | Description |
|---|---|
| Right to know | What data is collected, used, shared |
| Right to delete | Request deletion of personal information |
| Right to opt-out | Opt out of sale/sharing of data |
| Right to non-discrimination | No penalty for exercising rights |
| Right to correct | Correct inaccurate information |
| Right to limit | Limit use of sensitive personal information |

## 3. Data Processing Agreements

### DPA Required Contents (GDPR Art. 28)

| Section | Content |
|---|---|
| Subject matter and duration | What processing, how long |
| Nature and purpose | Why data is processed |
| Types of personal data | Categories of data |
| Categories of data subjects | Whose data |
| Controller obligations | Instructions, audit rights |
| Processor obligations | Security, sub-processors, assistance |
| Sub-processors | List, notification, liability |
| International transfers | Transfer mechanisms (SCCs, adequacy) |
| Data breach notification | Timeline (72 hours), content |
| Return/Deletion | What happens at contract end |

## 4. Privacy by Design

### Seven Principles

| Principle | Description | Implementation |
|---|---|---|
| Proactive not reactive | Prevent privacy issues | Privacy impact assessments |
| Privacy as default | Maximum privacy without action | Opt-in, minimal collection |
| Privacy embedded | Built into systems design | Architecture reviews |
| Full functionality | No false tradeoffs | Privacy AND security AND usability |
| End-to-end security | Lifecycle protection | Encryption, retention, deletion |
| Visibility and transparency | Open and accountable | Privacy notices, audits |
| User-centric | Respect user interests | Easy controls, clear choices |

### Data Protection Impact Assessment (DPIA)

```
When required:
- Systematic profiling with significant effects
- Large-scale processing of sensitive data
- Systematic monitoring of public areas
- New technologies with high risk

DPIA contents:
1. Description of processing
2. Assessment of necessity and proportionality
3. Assessment of risks to individuals
4. Measures to mitigate risks
5. Approval/sign-off
```

## 5. International Data Transfers

### Transfer Mechanisms

| Mechanism | Description | Status (2024-2025) |
|---|---|---|
| Adequacy decision | Country deemed adequate by EU | US (DPF), UK, Canada, Japan, etc. |
| Standard Contractual Clauses (SCCs) | EU-approved contract terms | Valid, requires TIA |
| Binding Corporate Rules (BCRs) | Intra-group transfers | Valid, complex to implement |
| EU-US Data Privacy Framework | US companies self-certify | Active since 2023 |
| Derogations | Consent, contract necessity | Limited use, case-by-case |

### Transfer Impact Assessment (TIA)

| Step | Activity |
|---|---|
| 1 | Map data flows and identify transfers |
| 2 | Identify transfer mechanism used |
| 3 | Assess destination country laws |
| 4 | Identify supplementary measures needed |
| 5 | Implement measures (technical, organizational) |
| 6 | Document assessment and re-evaluate periodically |
